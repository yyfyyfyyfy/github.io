踩点和查点

发表于 2016-07-18   |  

踩点

对于白帽子来说，在开始真正渗透之前，必须完成渗透前的信息收集，我们称之为踩点，踩点有很多手段，我们介绍两种。

网络共享主机（旁注）

旁注其实就是从旁注入的意思，是渗透入侵常用的一种手段。

通常，一台服务器上有很多站点，这些站点之间没有必然的联系，是相互独立的，使用的是不同的域名（甚至端口都不同），但是它们却共存在一台服务器上。

旁注就是利用同一主机上的不同网站的漏洞得到webshell，从而利用主机上的程序或者服务所暴露的用户的物理路径进行入侵，网络共享主机是更为严谨的学术叫法。

如果我们要对某一个站点进行直接渗透无法突破时，那么我们可以对同服务器的其他站点进行渗透，只要能打破一个缺口，就能沦陷服务区上的整个站点，甚至是一个集群

C段注入

C段注入是一个非常容易理解的渗透方式。首先，什么是C段呢？

比如在：111.111.111.14 这个IP上面有一个网站 。111.14 这个服务器上面有网站我们可以想想..他是一个非常大的站几乎没什么漏洞！但是在他同C段 111.111.111.1~111.111.111.255 这 1~255 上面也有服务器而且也有网站并且存在漏洞,那么我们就可以来渗透 .
1~255任何一个站 ，之后提权来嗅探得到111.14 这台服务器的密码， 甚至3389连接的密码和后台登录的密码 ，如果运气好会得到很多的密码…

那么，旁注和C段入侵哪个更理想呢？
旁站入侵更为理想。因为C段嗅探还涉及到提权，ARP。还要等待管理员登陆，才能抓取到密码。其实还有一个不用等待的方法，但那就涉及到社工方面了。

踩点有多种手段，这里只介绍旁注和Ｃ段注入两种，踩点其实就是获取目标的信息，越多越好，还可以通过whois服务,查找二级域名，ＣＤＮ后的真实ｉｐ，IDC机房ISP信息等踩点！

查点

黑客已经使用踩点技术成功地识别出了网站域名等目标范围清单，接下来他们通常会针对已知的弱点，对目标的 Web Server，服务器语言、开源程序名单/框架、敏感文件或功能点等网站技术整体做一个详细更为充分的探查。这个过程称之为 Web 查点。

那我们为什么要查点呢？

黑客已经使用踩点技术获得目标具体所使用的各种技术框架，可以精确获得目标现存的甚至未来所存在的隐患，例如发现目标所用论坛为 discuz 3.0 版本号，根据公开漏洞，可以直接进行攻击，如果发现目标使用版本较为安全，则不做无用功去扫描引起目标安全监控报警。

其过程类似高明的撬锁专家识别锁的安全级别类型,遇到高安全级别锁放弃另寻它法而不是盲目死磕。
常用手段：
1.Web 开源程序/框架查点。
2.Web 语言调试信息&Web Server 错误信息。其中有 Web 语言调试信息 phpinfo 页面，PHP 探针页面，Web Server IIS 报错。
3.Web 语言技术：PHP/ASP.NET/JSP 等。
4.操作系统：Windows 或 Linux 等操作系统发行版本号。
5.敏感文件等。
